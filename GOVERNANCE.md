# Governance

## Project Governance Model

This project follows an **owner-maintained** governance model.

### Maintainer

**Varun Kulkarni** ([@varunk130](https://github.com/varunk130)) — Project owner, primary maintainer.

### Decision Making

- **Technical decisions**: Made by the maintainer with community input via Issues and Discussions
- **Feature requests**: Evaluated based on alignment with project goals, complexity, and community interest
- **Security issues**: Handled with highest priority per [SECURITY.md](./SECURITY.md)

### Contribution Review

All contributions are reviewed by the maintainer for:

1. **Code quality** — Clean, readable, well-structured code
2. **Security** — No secrets, no real data, no vulnerabilities
3. **Alignment** — Fits the project's architecture and design philosophy
4. **Documentation** — Changes are documented appropriately

### Confidentiality

- All data in this project is synthetic and fictional
- Contributors must not introduce real customer data, proprietary information, or confidential business metrics
- Any contribution containing real data will be rejected and the contributor notified

### Intellectual Property

- All contributions are licensed under the MIT License
- Contributors retain copyright but grant the project a perpetual, irrevocable license per MIT terms

### Versioning

This project follows [Semantic Versioning](https://semver.org/):
- **Major**: Breaking changes to pipeline architecture or agent contracts
- **Minor**: New agents, features, or significant enhancements
- **Patch**: Bug fixes, documentation updates, style changes

## Code of Conduct

All participants must follow the [Code of Conduct](./CODE_OF_CONDUCT.md).
