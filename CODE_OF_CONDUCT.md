# Code of Conduct

This project welcomes contributors of all backgrounds and skill levels. We're committed to keeping the community respectful and productive.

## Expected behavior

- Be respectful in conversations, reviews, and issues.
- Assume good intent — clarify before reacting.
- Focus feedback on the work, not the person.
- Be patient with newcomers; everyone was new once.
- Credit prior work and collaborators.

## Unacceptable behavior

- Personal attacks, harassment, or discrimination of any kind.
- Sustained disruption of discussions, issues, or pull requests.
- Sharing other people's private information without consent.
- Any conduct that would reasonably be considered inappropriate in a professional setting.

## Enforcement

Maintainers are responsible for clarifying and enforcing this Code of Conduct. They may:

- Edit or remove comments, commits, code, issues, or pull requests that violate this Code.
- Temporarily or permanently ban contributors for repeated or severe violations.
- Decline to engage with conversations that have become unproductive.

Maintainers will explain enforcement decisions when asked.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please contact the maintainer through the contact details on their GitHub profile. Reports will be handled discreetly.

When reporting, please include:

- A description of the incident
- Where it happened (issue, PR, discussion link if available)
- Any other context that would help

## Scope

This Code of Conduct applies to all project spaces — issues, pull requests, discussions, code reviews, and any official communication channels — and to any individual representing the project in public spaces.

## Acknowledgment

This document is a short summary of community norms shared by many open source projects. It is not a substitute for legal advice or formal HR policies.
